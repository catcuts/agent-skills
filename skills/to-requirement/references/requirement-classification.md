# 需求分类体系

本文档说明功能级需求和系统级需求的区分标准。

## 功能级需求 vs 系统级需求

### 功能级需求 (Feature-Level Requirements)

**定义**: 用户可以直接操作或感知的具体功能点。

**特征**:
- 粒度较小，通常是一个独立的交互
- 可以用"用户可以 XXX"来描述
- 通常对应一个或几个 UI 组件
- 实现相对简单，可能只需几小时到几天

**判断标准**:
```
能否在 1-2 个工作日内完成？ → 可能是功能级需求
是否有独立的用户界面？ → 可能是功能级需求
用户能否直接操作？ → 可能是功能级需求
```

**示例**:
| 需求描述 | 类型 | 理由 |
|---------|------|------|
| 用户可以登录 | 功能级 | 直接操作，UI 明确 |
| 支持搜索功能 | 功能级 | 直接操作，UI 明确 |
| 用户可以导出 CSV | 功能级 | 直接操作，实现简单 |
| 支持夜间模式 | 功能级 | UI 配置项 |
| 用户可以上传头像 | 功能级 | 直接操作，实现简单 |

---

### 系统级需求 (System-Level Requirements)

**定义**: 架构层面的子系统或模块，包含多个功能，有明确的职责边界。

**特征**:
- 粒度较大，是一个完整的服务或模块
- 包含多个功能级需求
- 可能有独立的数据存储
- 通常需要几天到几周实现
- 可以独立部署或作为独立服务存在

**判断标准**:
```
是否包含多个子功能？ → 可能是系统级需求
是否有独立的数据模型？ → 可能是系统级需求
能否作为独立服务部署？ → 可能是系统级需求
需要 1 周以上时间实现？ → 可能是系统级需求
```

**示例**:
| 需求描述 | 类型 | 理由 |
|---------|------|------|
| 用户认证系统 | 系统级 | 包含登录、注册、密码重置等多个功能 |
| 支付系统 | 系统级 | 包含支付、退款、账单等多个功能 |
| 搜索引擎 | 系统级 | 包含索引、查询、排序、分词等 |
| 消息通知系统 | 系统级 | 包含邮件、短信、站内信等 |
| 数据分析平台 | 系统级 | 包含埋点、统计、报表、可视化等 |

## 层次关系

系统级需求可以分解为多个功能级需求：

```
[系统级] 用户认证系统
├── [功能级] 用户登录
├── [功能级] 用户注册
├── [功能级] 密码重置
├── [功能级] 第三方登录
├── [功能级] 会话管理
└── [功能级] 权限控制

[系统级] 支付系统
├── [功能级] 创建订单
├── [功能级] 支付处理
├── [功能级] 退款功能
├── [功能级] 发票管理
└── [功能级] 交易记录

[系统级] 搜索系统
├── [功能级] 关键词搜索
├── [功能级] 高级筛选
├── [功能级] 搜索建议
├── [功能级] 搜索历史
└── [功能级] 结果排序
```

## 边界情况处理

### 难以判断的情况

有些需求介于功能和系统之间，处理原则：

1. **看复杂度**: 如果一个功能实现复杂（涉及多个模块协作），提升为系统级
2. **看独立性**: 如果可以独立部署或作为第三方服务，视为系统级
3. **看上下文**: 根据项目规模调整判断标准

**示例**:

| 需求 | 小项目 | 大项目 | 理由 |
|-----|-------|-------|------|
| 文件上传 | 功能级 | 系统级 | 大项目可能需要独立的文件服务 |
| 评论功能 | 功能级 | 系统级 | 大项目可能需要独立的评论系统 |
| 日志记录 | 功能级 | 系统级 | 大项目可能需要独立的日志服务 |

### 处理建议

1. **优先识别系统级需求** - 先划分大模块
2. **再分解功能级需求** - 细化每个模块的功能
3. **保持灵活性** - 允许在实现过程中调整分类
4. **文档化** - 记录分类理由，便于后续评审

## 需求 ID 编码规则

为了便于追踪和关联，使用统一编码：

```
REQ-{类型}-{序号}

类型编码:
- F: 功能级 (Feature)
- S: 系统级 (System)
- T: 技术维度 (Technical)
- U: 用户体验维度 (UX)
- B: 商业价值维度 (Business)
- R: 风险合规维度 (Risk)
- O: 运维支撑维度 (Ops)
- E: 扩展性维度 (Extensibility)
```

**示例**:
- REQ-F001: 用户登录（功能级）
- REQ-S001: 认证系统（系统级）
- REQ-T001: JWT 认证（技术维度）
- REQ-U001: 记住我功能（用户体验维度）

**关联关系**:
```markdown
**关联需求**: REQ-S001 (属于认证系统)
```
